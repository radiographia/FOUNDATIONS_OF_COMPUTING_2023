Microbrowser‌

With the proliferation of mobile devices, such as mobile phones, Personal Digital Assistants (PDAs) and two-way pagers, several vendors are now producing Internet-enabled hand-held devices. These devices allow users to access e-mail and web pages on their mobile devices. Microbrowsers are browsers for Internet-enabled mobile devices. They can work with small, resource-restricted hand-held devices (having limited processing and memory resources), but offer reduced functionality than desktop browsers.

While desktop browsers occupy around 20-100 Mbytes, most microbrowsers occupy around 64 Kbytes to 8 Mbytes. Microbrowser designers achieve these small sizes by providing simplified interfaces and by eliminating much of desktop browser's multimedia functionality such as streaming-audio and streaming-video support.

Microbrowsers take one of the following two approaches to access web data:

1. Direct access approach. Microbrowsers based on this approach use an HTML-based language, such as compact HTML and XHTML. Compact HTML is a subset of HTML, which works with small devices' resource limitations by not supporting, for example, JPEG images, tables, multiple fonts, or more than two colors. XHTML is HTML reformulated as an XML application. XHTML lets a small device access just a subset of a web page for easier downloading and display.

In case of microbrowsers that use this approach, clients get data directly from a web server without any intervention [Figure 13.3(a)]. However, in this approach a web server can send information in HTML only.

A few microbrowsers that use this approach are Microsoft's Mobile Explorer, Opera Browser, and Pumatech's Browse-it.

2. WAP gateway approach. Microbrowers based on this approach use WAP's Wireless Markup Language (WML). As Figure 13.3(b) shows, this approach uses a content- adaptation server, known as WAP gateway. Any data downloaded from a web server first goes to the WAP gateway, which translates the data written in HTML or other languages into WAP's WML. Hence, the WAP gateway, which acts as a proxy, translates and formats the web content in WML for the target hand-held device./p>

A few microbrowsers that use this approach are Neomar Microbrowser and Openwave Mobile Browser.
