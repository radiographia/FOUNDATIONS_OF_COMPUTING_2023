Types of Firewalls‌

Commonly used firewalls are of the following types:

1. Packet-filtering router. Internet traffic flows in the form of IP packets. A packet-filtering router examines each incoming/outgoing packet and then forwards/discards the packet based on a set of packet-filtering rules defined by the security policies. Filtering rules use information in an IP packet to implement such policies as:

a. Allow/disallow a packet based on its source IP address.

b. Allow/disallow a packet based on its destination IP address.

c. Allow/disallow a packet based on its source and destination port number (transport- level address), which defines applications such as SNMP or TELNET.

d. Allow/disallow a packet based on its IP protocol field value (transport protocol).

Packet-filtering rules also define default action, which the packet-filtering router exercises when it does not find a direct rule match for a packet with any of the explicit packet- filtering rules. The two default policies are:

a. Default-deny. This policy discards all packets, which explicitly defined rules do not permit. It is a conservative policy and requires addition of every rule on a case-to- case basis for permitting different types of packets. However, it provides increased security.

b. Default-permit. This policy forwards all packets for which there are no explicitly defined rules. It is a liberal policy and has better ease of use for end users. However, it provides reduced security.

2. Proxy server. A proxy server is a firewall that examines and filters application-layer traffic. It is also called application-level gateway. When a TCP/IP application (such as HTTP, FTP, or Telnet) on a host on one network tries to communicate with a remote host on another network, the proxy server asks the user to specify a valid user ID and authentication information for accessing the application on the remote host. The proxy server contacts the application on the remote host with this information. If the proxy server does not implement the proxy code for a specific application, the security policy does not support this application service. Hence, the proxy server does not forward requests for this application across the firewall. A network administrator can also configure a proxy server to support only specific features of an application that the security policy considers acceptable while denying all other features.

Proxy servers are more secure than packet-filtering routers because they only examine a few allowable applications. With proxy servers, it is also easy to log and audit all incoming traffic at application level.

3. Connection-establishment server. A connection-establishment server is a firewall that establishes only permitted TCP connections between a user on the trusted network and a user on the untrusted network. It does not allow direct end-to-end TCP connection between two such users. Rather, it sets up two TCP connections, one between itself and a TCP user on the trusted network, and one between itself and a TCP user on the untrusted network. It finally connects the two TCP connections, if the security policy permits such a connection. Once the connection-establishment server establishes a connection (circuit) between the two TCP users, it relays TCP segments from one connection to the other without examining the contents. A connection-establishment server is also known as a circuit-level gateway.

4. Network Address Translating (NATing) server. NATing sever is a firewall that hides true IP addresses of critical hosts in the trusted network. Hosts protected by a firewall commonly use IP addresses in the “private address range”. Some of these hosts are critical and their security policy requires that the outside world should not know their IP addresses. In such a situation, an organization uses a NATing firewall, which uses a proxy IP address (known to the outside world) for such a host and translates the proxy IP address to the host's true IP address or vice-versa for all communications between the host and the outside world.

Note that the functionalities offered by different types of firewalls discussed above need not necessarily be on different physical machines. A properly configured single system can offer all these functionalities.
