What is IPSec?‌

The Internet Protocol (IPv4) did not have in-built security features. Hence, to enable secure communication in an IP-based application, application designers had to build security features into the application (applications had to carry out encryption/decryption and integrity checks in the application layer). That is, a source process had to encrypt and/or do integrity protection of the data before sending it to a destination process, which had to decrypt and/or carry out integrity verification. This approach had the following drawbacks:

1. It required changing all security-ignorant applications to make them security aware. This is a difficult task.

2. It required application programmers to understand security mechanisms and be able to use them correctly. Many application programmers do not have this capability.

To overcome these drawbacks, researchers proposed the idea of adding security features at

network layer. IPSec ( IP Security) is the security standard designed based on this idea.
