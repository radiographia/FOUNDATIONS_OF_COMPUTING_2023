Example of Message Transfer in the OSI Model‌

To illustrate the functions of various layers of the OSI model, let us consider a simple example of message transmission. With reference to Figure 12.23, let us assume that a process at sender node wants to send a message M to a process at receiver node. Sender node's process builds the message M and passes it to application layer (7) on its node. Application layer software adds a header ( H7) to M and passes the resulting message to presentation layer (6), via an interface between layers 7 and 6. Presentation layer software performs text compression, code conversion, security encryption, and so on, on the received message, and after adding a header (H6) to it, it passes the resulting message on to session layer (5). Depending on the type of dialog, session layer software establishes a dialog between sender and receiver processes. It also regulates direction of message flow. It adds a header (H5) to the message, and passes on the resulting message to transport layer (4). Transport layer software now splits the message into smaller units (M 1 and M2), called packets, and adds a header (H4) to each packet. These headers contain sequence numbers of these message packets. It then passes on these packets to network layer (3). Network layer software makes routing decisions for the received packets, and sets up a logical path between sender and receiver nodes for transmission of these packets. It then adds a header (H3) to each packet and passes them on to data-link layer (2). Data-link layer software adds a header (H2 ) and a trailer (T2) to each of these packets. The trailers contain checksum of data in corresponding packets. The resulting message units are called frames. Data-link layer passes on the frames to physical layer (1). Physical layer software transmits raw bits from sender node to receiver node by using the physical connectivity link between them.


Figure 12.23. An example illustrating transfer of message M from sender node to receiver node in the OSI model: Hn, header added by layer n: Tn, trailer added by layer n. On receiver node, message data traverses up from physical layer to application layer. As it traverses to higher-level layers, each layer performs its own functions, and strips off headers or trailers added by its peer layer at sender node. For example, data-link layer at receiver node performs error detection by recalculating checksum for each frame and comparing it with the checksum in the trailer of the frame. It strips off header (H2) and trailer (T 2) from the frames, before passing them on to network layer. Finally, application layer of receiver node passes on the message in its original form to communicating process on receiver node.

Notice that software of a particular layer on sender node conceptually communicates only with its peer layer on receiver node, although physically it communicates only with adjacent layers on sender node. This abstraction is crucial to network design.
