Proxy Server‌‌

An organization's network interconnects all computers of the organization. If the organization connects all these computers to the Internet directly, maintaining security becomes difficult for the organization because the Internet is an open environment. There may be virus attacks on organization's computing resources and other security problems through the Internet. To deal with this problem, an organization often uses a proxy server, which is a specially configured computer, which allows other computers of the organization to interact with the Internet through it. When only a single proxy server interacts with the Internet, the organization can easily maintain the security of its computing resources, because it can keep the single proxy server more secure than hundreds of individual computers on a network.

With proxy server, a user (employee) accesses the Internet from within the organization in the following manner. The user specifies the URL of a server on the Internet using the browser on his/her computer. The user's system sends the request to the proxy server, which then contacts the server on the Internet to obtain the requested information. The proxy server then sends the information received from the Internet server to the user's computer. All this is transparent to the user. That is, the user gets a feeling that he/she is accessing the Internet server directly. By acting as an in-between system, the proxy server can maintain security as well as log of all traffic between the Internet and the organization's network of computers.

Typically, an organization uses a proxy server for one or more of the following services:

1. To serve as a firewall. A firewall controls flow of messages to/from an organization from/to the Internet (outside world) for providing better security to the organization. For example, an organization can configure its firewall software running on proxy server for

packet filtering to ensure that the firewall permits only certain packets to flow from inside the organization to outside world, or vice-versa. Similarly, the organization can configure its firewall software to allow telnet to a computer within the organization only from those computers of the outside world, whose IP addresses have been pre-registered in the proxy server. Many firewalls also permit establishment of port filters on a per host or subnet basis, allowing the network administrator to control access on a fine grained basis than just the whole organization.

2. To log the Internet traffic. An organization can configure its proxy server to log every IP address, date and time of access, URL, number of bytes downloaded, and so on. It can use this information to analyze any attacks against the organization's systems. It can also use this information to analyze which employee spends how much time on the Internet and what types of information he/she accesses.

3. To scan viruses entering through the Internet. An organization can configure its proxy server to scan all incoming messages for potential virus attacks through the Internet. This protects the organization's computers from virus attacks because many computer viruses penetrate an organization's systems via the Internet.

4. To cache recently accessed web pages. Proxy servers can also speed up performance of some Internet services by caching recently accessed web pages on its local hard disk. If a user within an organization makes a request for a web page, which already exists in the cache of the organization's proxy server, the proxy server sends the web page to the user's computer directly from its cache rather than obtaining the web page from the specified Internet server and then forwarding it to the user's computer. This leads to faster Internet access in many situations.
