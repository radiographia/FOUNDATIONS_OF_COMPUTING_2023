Concept of Layered Protocols in Network Design‌

Modern computer networks have modular design for easy and efficient handling of the system. They consist of several modules, which are logically grouped into layers. Each layer offers certain services to higher layers, shielding those layers from implementation details of services offered by lower layers. Each layer has its own set of protocols. A particular layer of one machine communicates only with the corresponding layer of another machine by using the protocols of this layer.

The main reasons for using layered protocols in network design are:

1. Protocols of a network are complex. Designing them in layers makes their implementation more manageable.

2. Layering of protocols provides well-defined interfaces between layers, so that a designer can partition and implement various functionalities independently, and can change the modules of one layer without affecting the modules of other layers. For example, if technology improves, a designer can change routing algorithm of a network control program without affecting the message sequencing modules located in another layer of network architecture.

3. Layering of protocols also allows interaction between functionally paired layers in different locations. This concept aids in permitting distribution of functions to remote nodes.

Protocol suite, protocol family, or protocol stack are terms used to refer to a collection of protocols (of all layers) of a network system.
