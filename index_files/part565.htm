Pretty Good Privacy (PGP)‌‌

PGP is an open source software package for e-mail security. It is freely available (including its source code) on the Internet. It is available for different platforms such as UNIX, Linux, Windows, and MacOS. It provides the following features:

1. Authentication of sender and receiver of an e-mail message through the use of digital signature.

2. Confidentiality (privacy) of an e-mail message through the use of symmetric block encryption.

3. Compression of an e-mail message for its economic storage or transmission through the use of ZIP compression algorithm.

4. E-mail compatibility to provide transparency of e-mail applications. For this, it allows conversion of an encrypted e-mail message to an ASCII string by using base64 conversion.

5. Segmentation and reassembly of a long e-mail message to address the issue of maximum message size limitations.

Let us assume here that sender (A) wants to send a secret e-mail message (M) to receiver (B). Also, assume that Sa and Pa are the secret (private) and public keys, respectively of A, while Sb and Pb are the secret (private) and public keys, respectively of B. Both A and B know each others public key. PGP works as follows (see Figure 15.15).

1. A prepares message M and invokes the PGP program on its computer.

2. PGP uses MD5 digest function to compute the digest D of M.

3. PGP encrypts D by using A's secret key (Sa) to obtain a cypher text C1 = E (D, Sa). PGP uses RSA encryption algorithm for this. When B gets the message, it can decrypt it with A's public key (that it knows) and verify the correctness of the digest. Even if an intruder acquires C1 and decrypts it with A's known public key, the strength of MD5 guarantees

that is would be computationally infeasible to produce another message with the same digest (D). Hence, an intruder cannot tamper with the contents of message M.

4. PGP now concatenates M and C1 into a single message M1, and compresses M1 by using ZIP program, to obtain M1Z. For this, it uses Ziv-Lempel compression algorithm.

5. PGP then prompts B to send some random input. It uses this input to generate a 128-bit key (K).

6. PGP then encrypts M1Z with key K to obtain a cipher text C2 = E (M1Z, K). For this, it uses a block cipher called IDEA (International Data Encryption Algorithm). Note that K

serves as a one-time encryption key for IDEA.

7. PGP also encrypts K with B's public key Pb to obtain a cipher text C3 = E (K, Pb).

8. PGP then concatenates C2, C3, and the identifier (ID) of A, and converts the concatenated string to base64, which encodes the data by mapping 6-bit blocks of input to 8-bit blocks of output, all of which are printable ASCII characters. Thus, the resulting message Mr

contains only letters, digits, and the symbols +, /, and =.

9. PGP of A's computer then sends Mr to the PGP of B's computer.

10. PGP of B's computer reverses the base64 encoding to obtainC2, C3, and ID of A from Mr.

11. It then decrypts C3 with B's secret key Sb to obtain K.

12. It then decrypts C2 with K to obtain M1Z.

13. It then decompresses M1Z to obtain M1.

14. It then separates M and C1, and decryptsC1 by using the public key of A (Pa) to obtain D.

15. It then applies the same digest function (MD5) on M to compute digest D1 of M. If D1 and D are the same, it knows that messageM is correct (has not been tampered) and that it has come from A.

16. It then stores message M in B's mailbox with a note about its correctness check.


Figure 15.15. Illustrating the functioning of PGP (Pretty Good Privacy).

Note that PGP uses public key encryption/decryption (RSA scheme) only to encrypt/decrypt the 128-bit MD5 digest and to encrypt/decrypt the 128-bit IDEA key. It encrypts/decrypts the actual message using IDEA block cipher. Hence, although RSA is slow, PGP uses it for encrypting/decrypting only 256 bits. Therefore, PGP provides security, compression, and digital signature of e-mail messages in an efficient manner.
