Points to Remember‌

1. Computer security deals with protecting various resources and information of a computer system against destruction and unauthorized access. Main goals of computer security are secrecy, privacy, authenticity, and integrity.

2. A total approach to computer security involves both external and internal security. External security deals with securing a computer system against external factors such as fires, floods, earthquakes, stolen disks/tapes, etc. The three main aspects of internal security are authentication, access control, and communication security.

3. An intruder is a person or program that tries to obtain unauthorized access to data or a resource of a computer system. An intruder may attack a computer system in many ways. These attacks are broadly classified into two categories - passive attacks and active attacks.

4. In a passive attack, an intruder somehow tries to steal unauthorized information from a computer system without interfering with the normal functioning of the system. Some commonly used methods of passive attack are browsing, leaking, inferencing, and masquerading.

5. Active attacks interfere with the normal functioning of a system and often have damaging effects. Computer virus, worm, and logic bomb are three commonly used types of active attacks. Active attacks associated with message communications are integrity attack, authenticity attack, denial attack, delay attack, and replay attack.

6. Cryptography is a means of protecting private information against unauthorized access in situations, where it is difficult to provide physical security. Two broad classes of cryptosystems are symmetric and asymmetric.

7. Authentication is the process by which a system verifies the identity of a user (person or program) before it permits the user to access a requested resource. A system's authentication mechanism protects the system (or some resource of the system) from an unauthorized user by verifying the user's identity.

8. Normally, computer systems need authentication mechanisms for user login authentication, one-way authentication of communicating entities, and two-way authentication of communicating entities.

9. Basic approaches to authentication are proof by knowledge, proof by possession, and proof by property. Proof-by-knowledge method based on passwords is the most widely used method for user login authentication.

10. Access control deals with the ways to prohibit a user/process from accessing such resources/information, for which he/she is not having access authorization. Access matrix model is the most popular model used in existing systems to implement access control.

11. In access matrix model, access rights of each subject to each object are defined as entries in a matrix, called access matrix. Access control lists (ACLs) is the method used most commonly for implementing an access matrix.

12. The concept of digital signature, which uses asymmetric cryptosystems, is the most commonly used method to handle the issue of message integrity in distributed systems.

13. A firewall is a computer or communication device, which restricts flow of information between two networks in either direction or both directions. It is configured to implement some security policy (set of rules), which defines what kinds of interactions (flow of information) are permitted between the two networks.

14. Packet-filtering router, proxy server, connection-establishment server, and Network Address Translating (NATing) server are some commonly used types of firewalls.

15. Software piracy deals with illegal copying and resale of software. A software pirate is anyone who breaks system security for stealing intellectual property in software on that system.

16. Three types of attacks that fall under software piracy are illegal copying, malicious reverse engineering, and tampering.

17. Security mechanisms to protect software piracy include hardware-assisted security, hardware dongles, obfuscation, digital watermarking, and tamper proofing.

18. E-mail security deals with secrecy and integrity of electronic mails exchanged using e- mail applications. Some commonly used mechanisms/tools for e-mail security are Pretty Good Privacy (PGP), Privacy Enhanced Mail (PEM), and Secure/Multipurpose Internet Mail Extensions (S/MIME).

19. A spam e-mail [also known as Unsolicited Bulk E-mail (UBE)] is an unsolicited (unwanted, not asked for) e-mail message, which its sender sends in large quantities to an indiscriminate set of e-mail users. The act of sending a spam e-mail is known as e-mail spamming and a person who performs such an act is known as an e-mail spammer.

20. IPSec (Internet Protocol Security) is a security standard that adds security features at network layer so that application designers need not build security features into an IP- based application.

21. Web security deals with mechanisms to deal with various types of security issues (integrity, confidentiality, denial of service, and authentication) faced in using the Web. SSL and TLS are general-purpose solutions for Web security, which implement security just above TCP within the TCP/IP protocol stack.

22. Secure Electronic Transaction (SET) is a set of security protocols and formats for ensuring secure credit/debit card transactions (also called electronic transactions) on an untrusted network, such as the Internet.

23. Certification Authority (CA) and Public Key Infrastructure (PKI) are two important concepts proposed by the Internet Engineering Task Force (IETF). They enable two entities in an open network to exchange public keys securely, when the two entities want to communicate securely using public-key cryptography.

24. Steganography is a technique of hiding secret messages within the code representing a standard computer file in such a way that normal users cannot make out the difference between a file without the hidden secret message and a file with the hidden secret message. Only communicating entities can make out this difference and extract the hidden secret message from such a file. The file could be that of any media such as word- processed text, static image, audio recording, or video.

25. Steganalysis deals with techniques to determine whether a file contains a steganographically-hidden message. It employs mathematical techniques or artificial neural networks to statistically scan digital files and then examine them for changes in pixel values to detect the likely presence of steganographically-hidden data.

26. Some design principles that designers should use as basic guidelines for designing secure systems are least privilege, fail-safe defaults, open design, security built in to the system, checking for current authority, easy to grant and revoke access rights, not to trust other parties, always ensuring freshness of messages, building of firewalls, cost effective, efficient, convenient to use, and right set of trade-offs.

27. Cyber law is a branch of law that deals with legal issues related to the use of computers and information technology.

28. Cyber forensics deals with challenges to acquire digital evidences available in cyber crimes, analyze them, and to present them before the court of law in a manner that is acceptable to court for making decisions to prosecute cyber criminals.

29. CERT stands for Computer Emergency Response Team. It is a nation's most trusted referral agency for responding to computer security incidents whenever they occur.
