Layered Structure‌

Most modern operating systems organize their components into a number of layers (levels), each built on top of lower layers. The bottom most layer (layer 0) is the hardware and the top most layer (layer n) is the user interface. The number of in-between layers and their contents vary from one operating system to another. We explain here how operating systems designers decide

this.

The main advantage of layered approach is modularity. Operating systems designers select the layers such that each layer uses the functions and services of its immediate lower layer. This approach simplifies the design and implementation of the operating system, because the implementers can implement each layer by using only the operations of its immediate lower level layer.
