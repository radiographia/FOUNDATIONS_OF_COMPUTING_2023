Secure/Multipurpose Internet Mail Extensions (S/MIME)‌

RFC 822 (Request For Comments 822) defines a format for text messages sent using e-mail. It is the standard for Internet-based text e-mails. It structures an e-mail as having an envelope and contents. The envelope consists of the information required to transmit and deliver the e-mail to its recipients, while the contents consists of the message data (object) to be delivered to the recipient(s).

MIME is an extension to RFC 822. These extensions primarily address some of the limitations of RFC 822, SMTP (Simple Mail Transfer Protocol), and some other mail transfer protocols used for e-mail. For example, SMTP/822 has the following limitations:

1. SMTP cannot transmit executable files or other binary objects. Users have to use other protocols (such as UNIX UUencode/UUdecode protocol) for converting binary files into text form for use with SMTP.

2. SMTP is limited to 7-bit ASCII. Hence, it cannot transmit text data containing local language characters.

3. SMTP servers may reject mail messages exceeding certain size limit.

4. SMTP has translation problem between ASCII and EBCDIC character codes because SMTP gateways that perform this translation do not use a consistent set of mappings.

5. SMTP gateways to X.400 electronic mail networks cannot handle non-textual data included in X.400 messages.

6. Some SMTP implementations have problems such as:

a. They delete, add, or reorder carriage return and linefeed characters.

b. They truncate or wrap lines having more than 76 characters.

c. They remove trailing white space (tab or space characters).

d. They convert tab characters into multiple space characters.

MIME includes the following elements to resolve the problems as those mentioned above in a manner that is compatible with RFC 822 implementations:

1. It adds five new message header fields to RFC 822 message header. These fields provide information about the message contents.

2. It defines a number of content formats, which standardize representations supporting multimedia e-mail.

3. It defines transfer encodings, which enable conversion of any content format into a form that the mail system cannot alter.

S/MIME is an extension to MIME for making e-mails secured. It is an Internet standard approach to e-mail security, which incorporates the same functionality as PGP. It provides authentication, data integrity, secrecy, and non-repudiation by using digital signatures and

encryption. It provides the following functions:

1. Enveloped data. It contains encrypted content of e-mail message having any type of content. It also contains encryption keys used for content encryption for one or more recipients of the e-mail.

2. Signed data. It forms a digital signature by computing the message digest of the e-mail content and then encrypting it with the private key of the sender. It then concatenates the content with the message digest, and encodes the result with base64 encoding. Only a recipient having S/MIME capability can view such a signed e-mail message.

3. Clear-signed data. As in signed-data case, in this case also it forms a digital signature of the e-mail content. However, in this case, it encodes only the digital signature with base64 encoding. Hence, in this case, recipients without S/MIME capability can view the message content, although they cannot verify the signature.

4. Signed and enveloped data. It allows nesting of signed-only and encrypted-only entities. This enables signing of encrypted data, and encryption of signed data or clear-signed data whenever required.
