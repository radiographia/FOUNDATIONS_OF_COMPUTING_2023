SAN (Storage Area Network)‌‌

In this method, one or more processors and one or more storage devices are interconnected, forming a dedicated network called SAN. SAN technology provides any-to-any connection for processors and storage devices on the same SAN (see Figure 5.29).

As compared to DAS, SAN has the following advantages:

1. It provides scalability, enabling dynamic configuration of a large number of processors and storage devices in any-to-any connection model.

2. It enables storage sharing among multiple processors. Essentially, it makes multiple storage devices on the same SAN to appear as a single pool of storage devices accessible to all processors on the SAN.

3. It enables easy-manageability by allowing management of all storage devices on a SAN from a single point of control.

4. It enables building of reliable storage solutions, which can tolerate failures of both processors and storage devices.

However, as compared to DAS, SAN has the following limitations:

1. Initial cost of implementation is higher for SAN than DAS.

2. A SAN requires specialized hardware and software to manage the SAN and provide many of its potential benefits.

3. Processors cannot access storage devices on another SAN, which is part of the same system (see Figure 5.29).
