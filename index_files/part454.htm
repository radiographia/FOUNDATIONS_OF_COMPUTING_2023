ROUTING TECHNIQUES‌

Store-and-forward methods use routing techniques to select a path, out of multiple paths that exist between a pair of source and destination nodes. For example, in Figure 12.15, there are two paths between nodes A and B (1-5-4 and 1-2-3-4), and any one of them may be used to transmit a message/packet from node A to B .

An efficient routing technique is crucial to the overall performance of a network. A good routing algorithm should make routing decisions quickly and should be easily implementable in hardware. Furthermore, the decision process should not require global state information of the network because such information gathering is difficult and creates additional network traffic.

Note that broadcast channel method does not need any routing technique because a sender of a message simply puts the message on the communication channel and the receiver takes it off from the channel. The method does not need to select a path for transmitting the message from the sender to the receiver.

Routing algorithms usually depend on the following two attributes:

1. Place where routing decisions are made, and

2. Time constant of the information, which an algorithm uses to make routing decisions.
