SSL Connection and Session‌

SSL connection and SSL session are two important SSL concepts, which are defined as follows:

1. SSL connection. A connection is a peer-to-peer relationship between two parties. It serves as a transport for a service between them. There may be multiple simultaneous connections between a pair of parties. Each connection is associated with a session and is defined by a set of parameters (see Figure 15.17). Connections are transient.

2. SSL session. A session is an association between a client and a server. It defines a set of cryptographic security parameters, which a pair of parties (client and server) can share for multiple connections between them. Each session has a number of states associated with it. A session state is defined by a set of parameters (see Figure 15.18). Handshake protocol (described later) creates a session.
