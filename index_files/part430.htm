Points to Remember‌

1. Data processing is a series of actions or operations that converts data into information. Hence, data is raw material of information, and just as a manufacturing process transforms raw materials into finished products, data processing transforms raw data into information.

2. A data processing system consists of resources such as people, procedures, and devices,

which it uses to process input data for producing desirable output.

3. The six levels of data aggregation, which standard data organization mechanisms use in data storage hierarchy of data processing consist of bit, character, field, record, file, and database.

4. Multiple related bits combined together form a character, multiple related characters combined together form a field, multiple related fields combined together form a record, multiple related records combined together form a file, and multiple related files integrated together form a database.

5. Two standard methods of organizing data for data processing applications are file-oriented approach and database-oriented approach.

6. File-oriented approach of organizing data organizes an application's data into one or more files, and the application program(s) processes the data stored in these files to generate the desired output. This approach provides a set of programs to facilitate users to organize, create, delete, update, and manipulate their files. All these programs together form a File Management System (FMS).

7. Database-oriented approach of organizing data integrates data from multiple related files in the form of a database having the following properties:

It provides greater query flexibility. It reduces data redundancy.

It solves data integrity (inconsistency) problem.

It makes data independent of application programs, enabling users to add, change, and delete fields from database without affecting existing programs.

It also includes data security features at database level, record level, and even at field level to provide greater flexibility of restricted data access.

This approach provides a set of programs to facilitate users to organize, create, delete, update, and manipulate data in a database. All these programs together form a Database Management System (DBMS).

8. File organization deals with physical organization of records of a file for convenience of their storage and retrieval. Three file organizations, which data processing applications use commonly are sequential, direct/random, and indexed sequential.

9. File utilities are routines, which perform generic operations on data files. Commonly used file utilities are sorting, searching, merging, copying, printing, and maintenance.

10. A database model defines the manner in which a database links its various data elements together. Four commonly used database models are hierarchical, network, relational, and object-oriented.

11. Four major components of a database management system (DBMS) are data definition language (DDL), data manipulation language (DML), query language, and report generator.

12. Database designers use data definition language ( DDL) to define the structure of a database. Database structure definition is known as database schema.

13. Data manipulation language (DML) includes commands to enable users to enter and

manipulate data in a database.

14. Query language enables users to define their requirements as queries for extracting information from a database.

15. Report generator enables a database user to define a report's format by specifying its layout.

16. To prevent unauthorized individuals from accessing the data in a database, DBMSs commonly support several security features including user authentication, access control, and maintaining a log of database accesses.

17. In a client-server database system, a computer system, called database server stores, and manages the database. It runs important modules of the DBMS. These modules, called server programs, deal with the maintenance and processing of data in the database. On the other hand, database clients are systems on the same network, which users use for accessing the data in the database. They serve as front-end systems to the database and run those modules (called client programs) of the DBMS, which facilitate specification of access requests.

18. A distributed database system seamlessly integrates multiple databases, which are on different computers. A DBMS for a distributed database manages the multiple distributed databases in such a manner that they appear as a single database to end users.

19. An active database system is capable of monitoring and reacting to specific circumstances of relevance to an application.

20. A multimedia database is a collection of multimedia data items, such as text, graphics, animation, audio, and video. A multimedia database management system is a set of programs to define, create, store, access, manage, and query multimedia databases.

21. Special features needed in a multimedia database management system for handling multimedia data include

Support for multimedia data storage and retrieval Support for multimedia data types

Support for multimedia data relationships Support for multimedia data modeling Support for multimedia data manipulation Support for multimedia data query and Support for multimedia user interface

22. The techniques and tools for extraction of knowledge (useful information) from large datasets to help decision makers make meaningful decisions are the subject of the field called data science.

23. The five distinct stages involved in data science are:

Data selection Data preprocessing

Data transformation

Data mining and

Data interpretation/evaluation

24. Data warehousing is the process of creating a repository of integrated data, which users can use for decision support and analysis. A data warehouse is the resulting repository of integrated data. It is a subject-oriented, integrated, time-variant, and non-volatile collection of data, which (with proper analysis) supports management's decision-making process.

25. Data mining is the process of identifying useful patterns from large databases or data warehouses. It is defined as the nontrivial process of extracting implicit, previously unknown, and potentially useful information from data.

26. Popular classes of algorithms for data mining are classification, clustering, association, and trend analysis.

27. Popular techniques for data mining are statistics, machine learning, database-oriented, neural networks, fuzzy sets, and visual exploration.
