Representation of Algorithms‌

Commonly used ways to represent an algorithm are:

1. As program

2. As pseudocode

3. As flowchart

4. As decision table

An algorithm represented in the form of a programming language is a program. Hence, any

program is an algorithm, but the reverse is not true.

Programmers often use flowcharts, pseudocodes, and decision tables for representing algorithms for their programs before coding them in programming languages. Subsequent sections describe these program-planning tools. To plan their programs, programmers generally select one of them depending on their choice.
