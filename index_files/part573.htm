SECURE SOCKET LAYER (SSL) AND TRANSPORT LAYER SECURITY (TLS)‌

Web security deals with mechanisms to deal with various types of security issues (integrity, confidentiality, denial of service, and authentication) faced in using the Web. SSL and TLS are general-purpose solutions for Web security, which implement security just above TCP within the TCP/IP protocol stack (see Figure 15.16). A designer may choose to implement Web security using SSL or TLS in one of the following two ways:

1. By providing SSL or TLS as part of the underlying protocol suite.

2. By embedding SSL or TLS in specific packages. For example, designers of several Web browsers and Web servers implement SSL or TLS in the browser/server itself.

The former approach is completely general-purpose solution because it is transparent and applicable to any Web application.


Figure 15.16. Location of SSL or TLS in TCP/IP protocol stack. SSL and TLS are general-purpose solutions for Web security (can be applicable to any Web application) by virtue of this location.

Netscape originally designed SSL, and later published its public-reviewed version (Version 3) as an Internet draft document. Subsequently, IETF reviewed SSLv3 and published TLS as a common standard. TLS is similar to and backward compatible with SSLv3. We first describe here the SSLv3 protocol and then present the main differences between SSLv3 and TLS.
