Access Control Terminologies‌

When dealing with access control in computer systems, it is customary to use the following terms:

1. Objects. An object is an entity, which needs controlled access. It may be either an abstract entity (such as a process, a file, a database, a semaphore, a tree data structure) or a physical entity (such as a CPU, a memory segment, a printer, a card reader, a tape drive, a site of a network).

Each object has a system-wide unique name. In addition, each object has a “type”

associated with it, which determines the set of operations that may be performed on it. For example, the set of operations possible on:

a. Objects of type “data file” may be Open, Close, Create, Delete, Read, and Write,

b. Objects of type “program file,” may be Read, Write, and Execute,

c. Objects of type “semaphore,” may be Up and Down, and

d. Objects of type “tape drive,” may be Read, Write, and Rewind.

2. Subjects. A subject is an active entity wishing to access and perform operations on objects. The system grants access authorizations for various objects to a subject. Examples of subjects are processes and users. Note that subjects are also objects since they also require protection. Therefore, each subject also has a unique name.

3. Protection rules. Protection rules define possible ways in which subjects and objects have permission to interact. Therefore, associated with each (subject, object) pair is an access right that defines a subset of the set of possible operations for the object type that the subject may perform on the object. The complete set of access rights of a system defines which subjects can perform what operations on which objects. At any time, this set defines the protection state of the system at that time. The exact manner in which a system imposes its protection rules to control a subject's access to various objects depends on the access control model used by the system. Access matrix is the most popular model used in existing systems. Access control mechanisms based on access matrix model are described here.
