Mechanisms to Protect Software Piracy‌

Some commonly used security mechanisms to protect software piracy are:

1. Hardware-assisted security. Several software vendors use hardware assisted security schemes to prevent illegal copying of their software. For example, when vendors sell/supply software on memory devices such as Pen drive, Digital memory card, etc., they often insert some secret code on the devices, which a copy program cannot copy along with software when an attacker makes illegal copy of the software. When a user executes the software, the software first checks for the secret code, and fails to execute, if it does not find the secret code on the media. Many of these schemes are trade secrets of software vendors.

2. Hardware dongles. A dongle is a device sold with a software application to protect it against piracy. It attaches to a computer's I/O port and the application sends a query to it at regular intervals. If the dongle does not respond, the application refuses to run. Hence, copied software fails to execute without the dongle.

3. Obfuscations. Obfuscation (making difficult to understand) is a scheme to protect software against reverse engineering techniques. In this scheme, the code of a programP is obfuscated (transformed) into a new program P1 such that

a. P1 has the same semantics (observable behavior) as P.

b. Understanding and reverse engineering P1 is more difficult and time consuming than understanding and reverse engineering P.

Code obfuscation is similar to code optimization, except that, with obfuscation, we

maximize obscurity while minimizing execution time, whereas, with optimization we just minimize execution time. Lexical transformations, control transformations, and data transformations are some commonly used techniques for obfuscation.

4. Digital watermarking. Digital watermarking is the process of embedding information (called watermark) into a digital data (such as image, audio, video, text, or program code) in such a way that the watermark is unobtrusive and inseparable from the digital data. If anyone copies the data, the copied data carries the watermark. Digital watermarking, when used with software, discourages theft of intellectual property in software, or when such theft occurs, it allows the software owner to prove ownership. The technique embeds a secret information W (a watermark) into a program P such that

a. W can be reliably embedded into P and extracted from it.

b. Embedding W into P is resilient to de-watermarking attacks.

c. Embedding W into P does not change any statistical properties of P, and does not affect the performance of P adversely.

d. W has a mathematical property that allows us to argue that its presence in P is the result of deliberate actions.

e. Software watermarks are of two types - static and dynamic. Static watermarks are stored in the application executable itself, whereas, dynamic watermarks are constructed at runtime and stored in the dynamic state of the program.

5. Tamper proofing. Tamper proofing is a technique to protect software from tampering attacks. The technique attaches tamper-proof code to a program in such a way that the code detects if the program has been altered and causes the program to fail when tampering is evident. Three ways to detect tampering are:

a. By examining the executable program itself to check if it is identical to the original one. The method often uses a message digest algorithm (such as MD5) to speed up the test.

b. By examining the validity of intermediate results produced by the program. This technique is known as program (or result) checking.

c. By encrypting the executable, thereby preventing anyone modifying it successfully, unless he/she is able to decrypt it. The method uses hardware means, obfuscation, or both to protect the decryption routines from reverse engineering.
