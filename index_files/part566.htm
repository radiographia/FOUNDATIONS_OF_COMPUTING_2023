Privacy Enhanced Mail (PEM)‌

Privacy Enhanced Mail (PEM) scheme, designed for adding privacy to Internet mail applications, is a good example of use of cryptography and digital signature techniques. PEM offers confidentiality, authentication, and message integrity. These features provide sufficient trust so that the general Internet users feel comfortable using the Internet for business correspondence and sending messages that contain sensitive information. End systems implement PEM completely at application level so that they can incorporate it on site-by-site or

user-by-user basis. This approach imposes no special requirements on message transfer systems at intermediate relay sites or endpoints. That is, network routers and mail relays treat PEM messages as an ordinary piece of mail. How PEM provides privacy to electronic mails is described here.

PEM assumes that the network is not trusted, but each user of PEM trusts his/her own local computer. Mail users obtain a public/secret key pair from a local PEM program and publish their public keys with their mail addresses. PEM program maintains a database of secret keys of its local users and public keys of remote users. Currently, PEM uses Rivest-Shamir-Adleman (RSA) algorithm to generate public/secret key pairs for users. PEM provides the following types of facilities:

1. Confidentiality. Sending a message - in encrypted form so that an intruder cannot read sensitive information within it.

2. Message integrity. Sending a digitally signed message so that a receiver can be sure that no one has changed the contents of the message.

Both facilities also possess an authentication feature because only a user having proper key can do encryption and decryption of the message or the digital signature.

Let us first see how PEM sends a secret message (M) for ensuring confidentiality (readers are encouraged to draw an illustration diagram similar to Figure 15.15 for operations on sender and receiver computers):

1. PEM program of sender's computer first generates a random secret key (K) and encrypts the message (M) by using this key to obtain a ciphertext C1 = E(M, K). Currently, PEM uses DES algorithm for this purpose. PEM program then encrypts the secret key (K) by

using the recipient's public key (say, Pr) to obtain a ciphertext C2 = E(K, Pr). It then sends

C1 and C2 to the recipient in a message m1 .

2. On receiving m1, PEM program of recipient's computer fetches the recipient's secret key (Sr) from its database and decrypts C2 by using Sr to obtain K. Now by using K, it decrypts C1 to obtain the original message M, which it then stores in the recipient's mailbox.

Notice that PEM scheme retains the efficiency of symmetric cryptography for bulk encryption, but avoids the need for a secure key distribution server.

Now let us see how PEM sends a signed message (M) for ensuring message integrity (readers are encouraged to draw an illustration diagram similar to Figure 15.15 for operations on sender and receiver computers):

1. PEM program of sender's computer computes digest (D) of message (M) by using a message digest function. It then encrypts digest (D) by using the sender's secret key(Ss) to obtain a ciphertext C1 =(D, Ss). It then sends the sender's ID, C1, and M to the recipient in a message m1.

2. On receiving m1, PEM program of recipient's computer fetches the sender's public key

(Ps) from its database and uses it to decrypt C1 to obtain the digest D. It then applies the

same message digest function to M and compares the result with D. If the two are equal, it considers message M correct; otherwise, it considers it incorrect. It then stores message M in recipient's mailbox with a proper note about the result of its integrity check.
