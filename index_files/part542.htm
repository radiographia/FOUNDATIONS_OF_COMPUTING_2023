Symmetric and Asymmetric Cryptosystems‌

There are two broad classes of cryptosystems, symmetric and asymmetric. In a symmetric cryptosystem, either both the encryption key (Ke) and decryption key (Kd) are the same or one is easily derivable from the other. Usually, a common key (K) is used for both enciphering and deciphering. For security, it is important that users of a symmetric cryptosystem can easily alter the key and only authorized users know the key. Symmetric cryptosystems are also known as shared-key or private-key cryptosystems. They are useful in those situations where the subsystems performing both encryption and decryption of information are trusted subsystems. For example, a password-based user authentication system often uses this scheme for saving passwords in encrypted form. When a user declares a password, the operating system uses an encryption key to encrypt the password before storing it internally. At the time of authentication of a user (identified by his/her user-ID), the operating system retrieves the encrypted password corresponding to the specified user-ID, decrypts it using the same key, and compares it with the password supplied by the user.

In an asymmetric cryptosystem, on the other hand, the decryption key (Kd) is not equal to the encryption key (Ke). Furthermore, it is computationally impractical to derive Kd from Ke. Hence, the system needs to keep only Kd secret, and makes Ke publicly known. Asymmetric cryptosystems are also known as public-key cryptosystems.

Public-key cryptosystems are computationally expensive. Hence, they are unsuitable for bulk data encryption. A typical use of a public-key cryptosystem is for establishing a secured communication channel (virtual connection) between two communicating entities (A and B) in a distributed system. Once the system establishes a secured communication channel between entities A and B, they can then exchange messages using a symmetric cryptosystem. Let us assume that A and B want to establish a connection between them for initiating message transfers using a symmetric cryptosystem whose key is K. Note that it is insecure to send key K over a normal communication channel for the purpose of sharing it with A and B. Therefore, the system first uses a public-key cryptosystem to establish a secured connection between A and B in the following manner:

1. Entity A posts encryption key (Ke) of a public-key cryptosystem on, say, an electronic bulletin board, so that it is obtainable by B.

2. Entity B uses A's public key to encrypt key K and transmits it to A.

3. Entity A decrypts B's message using decryption key (Kd) of the public-key cryptosystem. Only A can decrypt this message because Kd is available with A only.

4. Now that A also has key K, both A and B can communicate safely with each other using the symmetric cryptosystem scheme. Any eavesdropper would only have public key (Ke), encrypted form of key K, and encrypted form of the messages exchanged between A and B.

One pitfall here is that someone else could masquerade as A or B. The system can overcome this problem by using digital signatures (described later in this chapter).

The relative advantages and disadvantages of symmetric and asymmetric cryptosystems are as follows:

1. A symmetric cryptosystem requires that only trusted subsystems should perform both encryption and decryption of information. An asymmetric cryptosystem does not require this. Therefore, general security policies need asymmetric cryptosystems.

2. When a communication system uses a symmetric cryptosystem for ensuring security of exchanged messages between two communicating entities, it requires a secure channel by which the sender can inform the receiver of the key, which they will be using to encrypt/decrypt messages. However, the system may transmit the encrypted messages through an insecure channel. On the other hand, in an asymmetric cryptosystem, the system can transmit both the public-key and the messages through an insecure channel. Therefore, there is no need for a special secure channel for key transmission. Hence, asymmetric cryptosystems are more secure than symmetric cryptosystems.

3. Normally, asymmetric cryptosystems are computationally much more expensive than symmetric cryptosystems. Hence, asymmetric cryptosystems are inefficient for bulk data encryption. In practice, therefore, data communication systems use asymmetric cryptosystems for initialization/control functions only, and symmetric cryptosystems for actual data transfer.

Data Encryption Standard (DES) is the best known and most widely used symmetric cryptosystem, whereas Rivest-Shamir-Adleman (RSA) is the first published and practically the most satisfactory asymmetric cryptosystem.
