Message Switching‌

A message is a logical unit of information of any length. In this method, the sender node of a message appends destination address to the message, and transmits it to its destination node either by store-and-forward or broadcast method.

Figure 12.17 illustrates the store-and-forward method, which transmits a message first from its source node to an intermediate node, which is connected to it directly. The intermediate node stores the message temporarily, inspects it for errors, and transmits it to one of its directly- connected nodes, based on an available free channel and its routing information. In this manner, each node that receives the message stores it temporarily for inspection, and forwards it to the next node until the message reaches its destination node. The actual path taken by a message to its destination is dynamic, because the method establishes a path for the message as it travels along. When the message reaches a node, the method releases the channel on which the message came so that the previous node can use it for transmission of another message. In Figure 12.17, if node A wants to transmit a message to node B, the message may take either path 1-2-3-4 or path 1-5-4, depending on the availability of path 1-2 or 1-5, when the message is at node 1.

Figure 12.18 illustrates the broadcast method, which broadcasts a message over a common medium known as broadcast channel. All nodes check the destination address attached to the message as it passes by, and ignore it if it does not belong to it. The destination node picks up the message when it reaches there. The method eliminates routing delays inherent in store-and- forward method. However, it requires that all nodes be simultaneously connected to the broadcast channel.


Advantages

1. The method does not require any physical connection between the source and destination nodes.

2. It makes effective use of channels, because it never blocks a channel unnecessarily for any user/node. It uses a channel whenever it has to transmit a message (of any user).


Disadvantages

1. Each node must have sufficient storage to buffer messages. In many cases, this storage space may be under-utilized because the message length is practically unlimited and each node must provision for the largest message.

2. In store-and-forward method, each intermediate node delays a message by the time it requires to handle the message. A node's message handling time includes the time it requires to receive the message, plus a queuing delay while waiting for an opportunity to retransmit the message to next node. On the other hand, in broadcast method, a node has to wait for sending its message until the broadcast channel becomes free.

3. The method is too slow for interactive/real-time applications. Hence, only those applications in which message delays are not critical use it.
