CLOUD COMPUTING‌

Due to the high cost involved in procurement, maintenance, and operation of computing resources, and fast obsolescence of these resources, many organizations and individuals now want to purchase computing resources as services and want to pay only for what they use. Cloud computing paradigm meets this requirement of users. It delivers all types of computing and communication resources as services including hardware resources (such as compute power engine and storage), software resources (such as software libraries, applications, and programming tools), and communication resources (such as communication services and collaboration tools). It enables users to access these resources on-demand using various types of access devices (such as PC, Workstation, Laptop, Smartphone, and Mobile phone). A user pays only for the type and amount of services he/she uses. A pool of servers of all types aggregated for building this service model is known collectively as a cloud (Figure 12.27), which is a metaphor for the Internet that is depicted in network diagrams as a cloud.

Cloud computing offers several advantages to end users such as:

1. No hardware acquisition costs

2. No software procurement or licensing upgrade costs

3. No operation cost for operating expensive servers

4. No need to invest for buying or leasing real estate for housing server systems

5. No hassles of upgrading resources due to fast obsolescence

6. No need to hire and maintain employees for maintenance and operation of expensive IT infrastructure

7. Pay only for the type and amount of services used


Figure 12.27. A cloud computing infrastructure.

An important advantage of cloud computing is that it hides the complexity of a cloud infrastructure from its end users. Users do not know or need to know what all is inside the cloud and how they work. They only care that the cloud delivers the services they need.

By its very nature, cloud computing architecture requires to support extremely high level of efficiency, service level availability of resources, scalability, manageability, security, and other qualities to seamlessly integrate its various components. Virtualization is the key design concept that cloud architectures use to achieve these objectives. Virtualization allows servers, storage devices, and other hardware and software resources to be treated as a pool of resources (rather than discrete systems), so that these resources can be shared and allocated on-demand. Cloud computing architectures support both para-virtualization and clustering. Para-virtualization allows a single server to be treated as multiple virtual servers so that a powerful server can be shared by and service multiple user requests simultaneously. Clustering allows multiple servers to be treated as a single server so that the power of multiple servers can be combined to service a single request needing a powerful server, which no single server has the capability to handle alone.

Google App Engine, Amazon Web Services, and Force.com are a few examples of cloud computing systems. Google App Engine services applications on Google's infrastructure.Amazon Web Services offers compute and storage services as EC2 and S3 respectively. Force.com offers application development environment as a service. It also provides necessary building blocks for an organization to build its own cloud computing system. Google's Apps, Maps, and Gmail are part of Google's cloud computing environment.
