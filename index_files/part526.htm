Temporal Specification‌

Temporal specification models temporal relationships among a set of related abstract multimedia objects of a presentation scenario, where an abstract object denotes a logical unit of media data such as a video or a text object. Two approaches in common use for temporal specification are graphical-based and language-based. They are described here.


Graphical-based Approach

This approach represents temporal relationships among various objects of a presentation scenario in the form of a graph, such as an X-Y graph with X-coordinate representing time and Y- coordinate representing objects, or a Petri net. O3, and O4. Si denotes the start point of object Oi. Figure 14.15 represents the same multimedia scenario as a petri net.

Language-based Approach

This approach represents temporal relationships among various objects of a presentation scenario in the form of a special programming language that has temporal synchronization functions. A few examples of synchronization functions that such a language may have are:

1. SyncAtomicSerial (A, B): Object B starts after object A stops.

2. SyncAtomicParallel (A, B): Objects A and B start simultaneously.

3. SynElemSerial (t1, A, t2, B): Object A starts after time t1, and object B starts after time t2

after A stops.

4. SyncElemParallel (t1, A, t2, B): Object A starts after time t1, and object B starts after time

t2. Based on the values of t1 and t2, objects A and B may overlap.
